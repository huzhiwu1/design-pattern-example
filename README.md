# design-pattern-example

> typescript 和 javascript 两个版本的设计模式

## [单一责任原则](./src/singleResponsibility)

> 每个类都应该有一个单一的功能
> 并且该功能应该由这个类完全封装起来

## [单例模式](./src/singleton)

> 定义：保证一个类仅有一个实例，并提供一个访问它的全局访问点

## [桥接模式](./src/bridge)

> 通过抽象将组件连接起来

> 桥接模式可以防止了“笛卡尔乘积”复杂性的爆炸
